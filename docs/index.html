01 The x402 flow

Payment becomes part
of the request.

No accounts, no billing dashboards, no human in the loop. The agent calls a paid endpoint, gets a price, pays on-chain, and has its data back — every payment a real transaction on Casper.

03 The registry

A full wallet, exposed
as eight MCP tools.

Self-custody and budget-limited. Your Ed25519 or secp256k1 key signs locally, nothing is sent to a backend.

IndexToolFunctionClass
01 x402_fetch Fetch any URL with automatic 402 handling — pay on-chain & retry transparently. x402
02 pay Settle an x402 payment on-chain and return the proof header to unlock the resource. x402
03 search_bazaar Discover x402-gated services the agent can pay for. x402
04 check_balance CSPR balance, public key, and account hash on the configured network. wallet
05 transfer_cspr Send native CSPR to any Casper account. wallet
06 transfer_token Send a CEP-18 token on Casper. wallet
07 spending_report Per-call and daily spend audit trail with budget remaining. wallet
08 request_funding Funding request with address + a testnet faucet hint. wallet

05 Quickstart

Connect it to Claude
in four steps.

MidOS runs as an MCP server. Point Claude Desktop or any MCP client at it, hand it a key, and your agent can start paying. No backend, no API keys.

1

Get a key and some testnet CSPR

MidOS is self-custody, it signs locally with an Ed25519 key you control. Bring your own, or generate one, then top it up from the testnet faucet.

terminal 
# generate an Ed25519 key (or use one you already have)
casper-client keygen ./keys
# fund the public key at the testnet faucet:
# https://testnet.cspr.live/tools/faucet
2

Add MidOS to your MCP config

Drop this into your Claude Desktop config. npx fetches and runs the published wallet no clone required. CASPER_SECRET_KEY takes hex, PEM contents, or a path to a .pem.

claude_desktop_config.json 
{
  "mcpServers": {
    "midos": {
      "command": "npx",
      "args": ["-y", "midos-wallet-mcp"],
      "env": {
        "CASPER_SECRET_KEY": "<ed25519 or secp256k1 key, hex or .pem>",
        "NETWORK": "casper-test",
        "CASPER_NODE_URL": "https://node.testnet.casper.network/rpc",
        "MAX_PER_CALL": "5.0",
        "MAX_PER_DAY": "50.0"
      }
    }
  }
}
macOS ~/Library/Application Support/Claude/claude_desktop_config.json
Windows %APPDATA%\Claude\claude_desktop_config.json
3

Restart Claude

Quit and reopen Claude Desktop. MidOS connects and exposes its eight wallet tools. Budgets are enforced before any signature a runaway loop can’t drain the key past MAX_PER_CALL or MAX_PER_DAY.

4

Just ask

That’s it talk to your agent in plain language. It discovers the price, checks budget, pays with a real on-chain transfer, and returns the data on its own.
